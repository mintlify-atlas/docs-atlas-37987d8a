## Overview

ClickHouse supports standard SQL DDL (Data Definition Language) and DML (Data Manipulation Language) statements with extensions for distributed systems and real-time analytics.

## CREATE Statements

### CREATE TABLE

Create a new table with specified engine and structure.

<CodeGroup>
```sql Basic Table
CREATE TABLE users (
    user_id UInt64,
    username String,
    email String,
    created_at DateTime
) ENGINE = MergeTree()
ORDER BY user_id;
```

```sql With Primary Key
CREATE TABLE events (
    event_id UInt64,
    user_id UInt64,
    event_type String,
    event_time DateTime,
    properties Map(String, String)
) ENGINE = MergeTree()
PRIMARY KEY (event_type, event_time)
ORDER BY (event_type, event_time, user_id);
```

```sql With Partitioning
CREATE TABLE logs (
    log_date Date,
    log_time DateTime,
    level String,
    message String
) ENGINE = MergeTree()
PARTITION BY toYYYYMM(log_date)
ORDER BY (log_date, log_time);
```
</CodeGroup>

#### Parameters

<ParamField path="IF NOT EXISTS" type="modifier">
  Create table only if it doesn't exist. Prevents errors when table already exists.
</ParamField>

<ParamField path="ENGINE" type="required">
  Table engine that defines storage and query behavior. Common engines: `MergeTree`, `ReplicatedMergeTree`, `Log`, `Memory`.
</ParamField>

<ParamField path="ORDER BY" type="required for MergeTree">
  Determines the sort order for data storage. Critical for query performance.
</ParamField>

<ParamField path="PARTITION BY" type="optional">
  Expression for partitioning data. Enables efficient data management and pruning.
</ParamField>

<ParamField path="PRIMARY KEY" type="optional">
  Primary key columns. Defaults to `ORDER BY` if not specified.
</ParamField>

<ParamField path="SAMPLE BY" type="optional">
  Expression for sampling. Must be included in primary key.
</ParamField>

### CREATE DATABASE

<CodeGroup>
```sql Simple Database
CREATE DATABASE analytics;
```

```sql With Engine
CREATE DATABASE IF NOT EXISTS my_db
ENGINE = Atomic;
```
</CodeGroup>

### CREATE VIEW

<CodeGroup>
```sql Normal View
CREATE VIEW active_users AS
SELECT user_id, username
FROM users
WHERE status = 'active';
```

```sql Materialized View
CREATE MATERIALIZED VIEW user_stats
ENGINE = SummingMergeTree()
ORDER BY user_id
AS SELECT
    user_id,
    count() AS event_count,
    sum(amount) AS total_amount
FROM events
GROUP BY user_id;
```
</CodeGroup>

## DROP Statements

### DROP TABLE

<CodeGroup>
```sql Drop Table
DROP TABLE users;
```

```sql Conditional Drop
DROP TABLE IF EXISTS users;
```

```sql No Delay
DROP TABLE users NO DELAY;
```
</CodeGroup>

### DROP DATABASE

```sql
DROP DATABASE IF EXISTS analytics;
```

### DROP VIEW

```sql
DROP VIEW IF EXISTS active_users;
```

## TRUNCATE Statement

Remove all data from a table:

```sql
TRUNCATE TABLE events;
```

## RENAME Statement

<CodeGroup>
```sql Rename Table
RENAME TABLE old_name TO new_name;
```

```sql Atomic Rename
RENAME TABLE
    table1 TO table1_old,
    table2 TO table2_old,
    table3 TO table1,
    table4 TO table2;
```
</CodeGroup>

## OPTIMIZE Statement

Trigger background merges:

<CodeGroup>
```sql Basic Optimize
OPTIMIZE TABLE events;
```

```sql Final Optimization
OPTIMIZE TABLE users FINAL;
```

```sql Deduplicate
OPTIMIZE TABLE users DEDUPLICATE;
```

```sql By Columns
OPTIMIZE TABLE users DEDUPLICATE BY user_id, email;
```
</CodeGroup>

## CHECK Statement

Verify table data integrity:

```sql
CHECK TABLE events;
```

## DESCRIBE Statement

<CodeGroup>
```sql Describe Table
DESCRIBE TABLE users;
```

```sql Show Create

```
</CodeGroup>


```sql Show Tables

```



## USE Statement

Set default database:

```sql
USE analytics;
```

## DELETE Statement

<Note>
DELETE is a lightweight operation in ClickHouse. For heavy deletions, use ALTER TABLE DELETE.
</Note>

```sql
DELETE FROM events
WHERE event_date < '2024-01-01';
```

## UPDATE Statement

<Warning>
UPDATE is not recommended for frequent operations. Use mutations sparingly.
</Warning>

```sql
ALTER TABLE users UPDATE status = 'inactive'
WHERE last_seen < now() - INTERVAL 90 DAY;
```

## KILL QUERY Statement

Terminate running queries:

<CodeGroup>
```sql Kill by Query ID
KILL QUERY WHERE query_id = 'query-id-123';
```

```sql Kill by User
KILL QUERY WHERE user = 'analytics_user' SYNC;
```
</CodeGroup>


</CodeGroup>

</CodeGroup>

## EXCHANGE Statement

Atomically exchange tables:

```sql
EXCHANGE TABLES table1 AND table2;
```

## SET Statement

Set session settings:

<CodeGroup>
```sql Single Setting
SET max_threads = 8;
```

```sql Multiple Settings
SET max_memory_usage = 10000000000,
    max_execution_time = 60;
```
</CodeGroup>

## EXPLAIN Statement

Analyze query execution:

<CodeGroup>
```sql Explain Plan
EXPLAIN SELECT * FROM users WHERE user_id > 1000;
```

```sql Explain Pipeline
EXPLAIN PIPELINE SELECT count() FROM events;
```

```sql Explain Syntax
EXPLAIN SYNTAX SELECT * FROM users WHERE user_id > 1000;
```
</CodeGroup>

## Best Practices

<Tip>
- Always use `IF EXISTS` / `IF NOT EXISTS` for idempotent operations
- Set appropriate `ORDER BY` for optimal query performance
- Use partitioning for time-series data
- Avoid frequent UPDATEs and DELETEs - design for append-only when possible
</Tip>

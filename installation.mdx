# Installation

Multiple installation methods are available for ClickHouse. Choose the one that best fits your environment.

## Quick Install (Linux, macOS, FreeBSD)

The fastest way to install ClickHouse:

```bash
curl https://clickhouse.com/ | sh
```

This script downloads the appropriate binary for your platform and makes it executable.

<Note>
This method works on Linux (x86_64, ARM64), macOS, and FreeBSD systems.
</Note>

## Docker Installation

Docker is the recommended method for development and testing.

### Start ClickHouse Server

```bash
docker run -d \
  --name some-clickhouse-server \
  --ulimit nofile=262144:262144 \
  clickhouse/clickhouse-server
```

<Warning>
By default, ClickHouse is accessible only via the Docker network. See the networking section below to expose ports.
</Warning>

### Connect Using Native Client

<CodeGroup>
```bash Via Docker Network
docker run -it --rm \
  --network=container:some-clickhouse-server \
  --entrypoint clickhouse-client \
  clickhouse/clickhouse-server
```

```bash Using Exec
docker exec -it some-clickhouse-server clickhouse-client
```
</CodeGroup>

### Connect Using HTTP Interface

Test connectivity with `curl`:

```bash
echo "SELECT 'Hello, ClickHouse!'" | \
  docker run -i --rm \
    --network=container:some-clickhouse-server \
    buildpack-deps:curl \
    curl 'http://localhost:8123/?query=' -s --data-binary @-
```

**Output:**
```
Hello, ClickHouse!
```

## Docker with Exposed Ports

To access ClickHouse from outside Docker, map the ports and set a password:

```bash
docker run -d \
  --name some-clickhouse-server \
  -p 18123:8123 \
  -p 19000:9000 \
  -e CLICKHOUSE_PASSWORD=changeme \
  --ulimit nofile=262144:262144 \
  clickhouse/clickhouse-server
```

Test the connection:

```bash
echo 'SELECT version()' | \
  curl 'http://localhost:18123/?password=changeme' --data-binary @-
```

<Note>
Ports:
- `8123`: HTTP interface for queries and integrations
- `9000`: Native client protocol (used by `clickhouse-client`)
</Note>

### Using Host Network

For better network performance, use host networking:

```bash
docker run -d \
  --network=host \
  --name some-clickhouse-server \
  --ulimit nofile=262144:262144 \
  clickhouse/clickhouse-server
```

<Warning>
With host networking, the `default` user is accessible only for localhost requests unless you set `CLICKHOUSE_PASSWORD`.
</Warning>

## Persistent Storage with Volumes

For production use, mount volumes to persist data:

```bash
docker run -d \
  --name some-clickhouse-server \
  -v "$PWD/ch_data:/var/lib/clickhouse/" \
  -v "$PWD/ch_logs:/var/log/clickhouse-server/" \
  --ulimit nofile=262144:262144 \
  clickhouse/clickhouse-server
```

Key directories:
- `/var/lib/clickhouse/` - Main data directory
- `/var/log/clickhouse-server/` - Server logs
- `/etc/clickhouse-server/config.d/*.xml` - Configuration overrides
- `/etc/clickhouse-server/users.d/*.xml` - User settings
- `/docker-entrypoint-initdb.d/` - Initialization scripts

## Custom Configuration

Mount a custom configuration file:

```bash
docker run -d \
  --name some-clickhouse-server \
  --ulimit nofile=262144:262144 \
  -v /path/to/your/config.xml:/etc/clickhouse-server/config.xml \
  clickhouse/clickhouse-server
```

## Environment Variables

Configure ClickHouse using environment variables:

### Create Default Database and User

```bash
docker run -d \
  --name some-clickhouse-server \
  -e CLICKHOUSE_DB=my_database \
  -e CLICKHOUSE_USER=username \
  -e CLICKHOUSE_PASSWORD=password \
  -e CLICKHOUSE_DEFAULT_ACCESS_MANAGEMENT=1 \
  -p 9000:9000 \
  clickhouse/clickhouse-server
```

### Managing the Default User

The `default` user has network access disabled unless you set one of:
- `CLICKHOUSE_PASSWORD`
- `CLICKHOUSE_USER`
- `CLICKHOUSE_DEFAULT_ACCESS_MANAGEMENT`

To enable the default user without authentication (not recommended for production):

```bash
docker run -d \
  --name some-clickhouse-server \
  -e CLICKHOUSE_SKIP_USER_SETUP=1 \
  -p 9000:9000 \
  clickhouse/clickhouse-server
```

<Warning>
Never use `CLICKHOUSE_SKIP_USER_SETUP=1` in production environments.
</Warning>

## Initialization Scripts

Run SQL scripts or shell scripts on first startup by placing them in `/docker-entrypoint-initdb.d/`:

```bash init-db.sh
#!/bin/bash
set -e

clickhouse client -n <<-EOSQL
    CREATE DATABASE docker;
    CREATE TABLE docker.docker (x Int32) ENGINE = Log;
EOSQL
```

Mount the script:

```bash
docker run -d \
  --name some-clickhouse-server \
  -v "$PWD/init-db.sh:/docker-entrypoint-initdb.d/init-db.sh" \
  clickhouse/clickhouse-server
```

<Tip>
Supported file types:
- `*.sql` - Executed as SQL
- `*.sql.gz` - Decompressed and executed as SQL
- `*.sh` - Executable scripts are run, non-executable scripts are sourced
</Tip>

## Linux Capabilities

For advanced functionality, enable Linux capabilities:

```bash
docker run -d \
  --cap-add=SYS_NICE \
  --cap-add=NET_ADMIN \
  --cap-add=IPC_LOCK \
  --name some-clickhouse-server \
  --ulimit nofile=262144:262144 \
  clickhouse/clickhouse-server
```

These capabilities enable:
- `SYS_NICE`: Thread priority management
- `NET_ADMIN`: Network configuration
- `IPC_LOCK`: Memory locking for performance

## Running as Custom User

Run ClickHouse as a specific user for proper file ownership:

```bash
# Create directories with proper ownership
mkdir -p "$PWD/data/clickhouse" "$PWD/logs/clickhouse"

docker run -d \
  --user "${UID}:${GID}" \
  --name some-clickhouse-server \
  --ulimit nofile=262144:262144 \
  -v "$PWD/logs/clickhouse:/var/log/clickhouse-server" \
  -v "$PWD/data/clickhouse:/var/lib/clickhouse" \
  clickhouse/clickhouse-server
```

<Note>
When mounting local directories, always specify `--user` to maintain proper file ownership.
</Note>

## Platform Compatibility

### AMD64 (x86_64)

Requires SSE3 instruction support. Virtually all x86 CPUs after 2005 support SSE3.

### ARM64 (AArch64)

Requires:
- ARMv8.2-A architecture
- Load-Acquire RCpc register support

Supported platforms:
- AWS Graviton 2+
- Azure ARM instances
- GCP ARM instances

Unsupported:
- Raspberry Pi 4 (ARMv8.0-A)
- Jetson AGX Xavier/Orin (ARMv8.2-A without RCpc)

### Docker Version Requirements

<Warning>
ClickHouse 24.11+ Ubuntu images require Docker >= 20.10.10. 

If using older Docker versions, either:
- Update Docker to >= 20.10.10, or
- Use `--security-opt seccomp=unconfined` (has security implications)
</Warning>

## Available Docker Tags

- `latest` - Latest release of the stable branch
- `22.2` - Latest release of branch 22.2
- `22.2.3`, `22.2.3.5` - Specific version tags
- `head` - Built from latest commit to default branch
- `*-alpine` - Alpine Linux based images (smaller size)

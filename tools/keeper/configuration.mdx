# ClickHouse Keeper Configuration

ClickHouse Keeper uses an XML configuration file, typically named `keeper_config.xml`. The configuration is loaded from `/etc/clickhouse-keeper/` or can be specified with the `--config-file` option.

## Basic Configuration

### Minimal Configuration

A minimal single-node Keeper configuration:

```xml
<clickhouse>
    <logger>
        <level>information</level>
        <log>/var/log/clickhouse-keeper/clickhouse-keeper.log</log>
        <errorlog>/var/log/clickhouse-keeper/clickhouse-keeper.err.log</errorlog>
        <size>1000M</size>
        <count>10</count>
    </logger>

    <keeper_server>
        <tcp_port>9181</tcp_port>
        <server_id>1</server_id>
        <log_storage_path>/var/lib/clickhouse/coordination/logs</log_storage_path>
        <snapshot_storage_path>/var/lib/clickhouse/coordination/snapshots</snapshot_storage_path>

        <coordination_settings>
            <operation_timeout_ms>10000</operation_timeout_ms>
            <session_timeout_ms>30000</session_timeout_ms>
        </coordination_settings>

        <raft_configuration>
            <server>
                <id>1</id>
                <hostname>localhost</hostname>
                <port>9234</port>
            </server>
        </raft_configuration>
    </keeper_server>
</clickhouse>
```

### Storage Paths

Configuration from `programs/keeper/Keeper.cpp:290-316`:

```xml
<keeper_server>
    <!-- Unified storage path (recommended) -->
    <storage_path>/var/lib/clickhouse/coordination</storage_path>

    <!-- Or separate paths for logs and snapshots -->
    <log_storage_path>/var/lib/clickhouse/coordination/logs</log_storage_path>
    <snapshot_storage_path>/var/lib/clickhouse/coordination/snapshots</snapshot_storage_path>
</keeper_server>
```

If `storage_path` is specified, logs and snapshots will be stored in subdirectories. Default path is `/var/lib/clickhouse/coordination/`.

### Server Identity

```xml
<keeper_server>
    <!-- Unique server ID (must be unique in cluster, required) -->
    <server_id>1</server_id>

    <!-- Super digest for admin access (format: user:base64_digest) -->
    <superdigest>admin:YourBase64DigestHere</superdigest>
</keeper_server>
```

## Cluster Configuration

### Multi-Node Cluster

Example 3-node cluster configuration:

```xml
<keeper_server>
    <tcp_port>9181</tcp_port>
    <server_id>1</server_id>
    <log_storage_path>/var/lib/clickhouse/coordination/logs</log_storage_path>
    <snapshot_storage_path>/var/lib/clickhouse/coordination/snapshots</snapshot_storage_path>

    <!-- Enable hostname sanity checks -->
    <hostname_checks_enabled>true</hostname_checks_enabled>

    <raft_configuration>
        <server>
            <id>1</id>
            <hostname>keeper1.example.com</hostname>
            <port>9234</port>
        </server>
        <server>
            <id>2</id>
            <hostname>keeper2.example.com</hostname>
            <port>9234</port>
        </server>
        <server>
            <id>3</id>
            <hostname>keeper3.example.com</hostname>
            <port>9234</port>
        </server>
    </raft_configuration>
</keeper_server>
```

**Notes:**
- `tcp_port` (9181) is for client connections
- `port` (9234) in `raft_configuration` is for inter-server Raft communication
- Each server needs a unique `server_id` matching one entry in `raft_configuration`
- Hostname can be IP address or DNS name

```xml
        <certificateFile>/etc/clickhouse-keeper/server.crt</certificateFile>

        <privateKeyFile>/etc/clickhouse-keeper/server.key</privateKeyFile>

        <verificationMode>none</verificationMode>
        
        <!-- Load default CA file -->
        <loadDefaultCAFile>true</loadDefaultCAFile>
        
        <!-- Cache sessions -->
        <cacheSessions>true</cacheSessions>
        
        <!-- Disable weak protocols -->
        <disableProtocols>sslv2,sslv3</disableProtocols>
        
        <!-- Prefer server cipher order -->
        <preferServerCiphers>true</preferServerCiphers>
    </server>
</openSSL>
```

Generate self-signed certificate:
```bash
openssl req -subj "/CN=localhost" -new -newkey rsa:2048 -days 365 \
  -nodes -x509 -keyout /etc/clickhouse-keeper/server.key \
  -out /etc/clickhouse-keeper/server.crt
```

## Configuration Validation

Validate configuration before starting:
```bash
clickhouse-keeper --config-file=/etc/clickhouse-keeper/keeper_config.xml --help
```

Test configuration loading:
```bash
clickhouse-keeper --config-file=/etc/clickhouse-keeper/keeper_config.xml --version
```

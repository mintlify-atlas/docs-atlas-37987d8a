This guide covers common ClickHouse issues, diagnostic approaches, and solutions for operational problems.

## Server Issues

### Server Won't Start

**Check logs:**
```bash
sudo tail -f /var/log/clickhouse-server/clickhouse-server.log
sudo tail -f /var/log/clickhouse-server/clickhouse-server.err.log
```

**Common causes:**

1. **Configuration error**
   ```bash
   # Test configuration
   clickhouse-server --config-file=/etc/clickhouse-server/config.xml --test
   ```

2. **Port already in use**
   ```bash
   # Check if ports are occupied
   sudo netstat -tlnp | grep -E '(8123|9000)'
   
   # Change ports in config.xml if needed
   ```

3. **Insufficient disk space**
   ```bash
   # Check disk space
   df -h /var/lib/clickhouse/
   
   # Clean up temporary files
   sudo rm -rf /var/lib/clickhouse/tmp/*
   ```

4. **Permission issues**
   ```bash
   # Fix ownership
   sudo chown -R clickhouse:clickhouse /var/lib/clickhouse/
   sudo chown -R clickhouse:clickhouse /var/log/clickhouse-server/
   ```

5. **Corrupted metadata**
   ```bash
   # Backup and check metadata
   sudo cp -r /var/lib/clickhouse/metadata /var/lib/clickhouse/metadata.backup
   
   # Check for corrupted .sql files
   sudo find /var/lib/clickhouse/metadata -name "*.sql" -exec cat {} \;
   ```

### Server Crashes

<Warning>
Server crashes require immediate investigation. Always check logs and create a backup before attempting fixes.
</Warning>

**Investigate crash:**
```bash
# Check system logs
sudo journalctl -u clickhouse-server -n 1000

# Check for OOM killer
sudo dmesg | grep -i "out of memory"
sudo dmesg | grep -i "clickhouse"

# Check core dumps (if enabled)
ls -lh /var/lib/clickhouse/cores/
```

**Common causes:**
- Out of memory (OOM)
- Disk full
- Hardware failures
- Bug in ClickHouse (upgrade to latest version)

## Query Performance Issues

### Slow Queries

**Identify slow queries:**
```sql
SELECT 
    type,
    event_time,
    query_duration_ms / 1000 AS duration_sec,
    formatReadableSize(memory_usage) AS memory,
    formatReadableQuantity(read_rows) AS rows_read,
    user,
    query
FROM system.query_log
WHERE query_duration_ms > 5000
  AND type = 'QueryFinish'
  AND event_date = today()
ORDER BY query_duration_ms DESC
LIMIT 20;
```

**Diagnose with EXPLAIN:**
```sql
-- Analyze query plan
EXPLAIN PLAN
SELECT * FROM large_table WHERE date = '2025-01-01';

-- Analyze with pipeline
EXPLAIN PIPELINE
SELECT * FROM large_table WHERE date = '2025-01-01';

-- Show actual execution statistics
EXPLAIN ANALYZE
SELECT * FROM large_table WHERE date = '2025-01-01';
```

**Common solutions:**

<Steps>

1. **Add appropriate indexes**
   ```sql
   -- Add skip index
   ALTER TABLE my_table ADD INDEX idx_column column TYPE minmax GRANULARITY 4;
   ```

2. **Optimize partition key**
   ```sql
   -- Check partition usage
   SELECT 
       partition,
       count() AS parts,
       sum(rows) AS rows
   FROM system.parts
   WHERE table = 'my_table' AND active
   GROUP BY partition;
   ```

3. **Adjust sorting key**
   ```sql
   -- Ensure ORDER BY matches sorting key for best performance
   -- Recreate table with better sorting key if needed
   ```

4. **Increase resources**
   ```sql
   -- Increase max_threads for the query
   SET max_threads = 8;
   
   -- Increase memory limit
   SET max_memory_usage = 20000000000;
   ```

</Steps>

### High Memory Usage

**Monitor memory:**
```sql
-- Current memory usage
SELECT formatReadableSize(value) AS memory
FROM system.asynchronous_metrics
WHERE metric = 'MemoryResident';

-- Per-query memory
SELECT 
    user,
    query_id,
    formatReadableSize(memory_usage) AS memory,
    query
FROM system.processes
ORDER BY memory_usage DESC;
```

**Solutions:**
```sql
-- Set global memory limit
-- In config.xml:
<max_server_memory_usage>50000000000</max_server_memory_usage>

-- Limit per-query memory
SET max_memory_usage = 10000000000;

-- Enable memory overcommit tracking
SET memory_overcommit_ratio_denominator = 1073741824;
```

## Data Issues

### Too Many Parts

Too many parts can severely degrade performance:

```sql
-- Check parts count
SELECT 
    database,
    table,
    count() AS parts
FROM system.parts
WHERE active
GROUP BY database, table
ORDER BY parts DESC;

-- Tables with too many parts
SELECT 
    database,
    table,
    count() AS parts
FROM system.parts
WHERE active
GROUP BY database, table
HAVING parts > 300;
```

**Solutions:**

<Steps>

1. **Trigger manual merge**
   ```sql
   OPTIMIZE TABLE my_database.my_table FINAL;
   ```

2. **Adjust merge settings**
   ```sql
   ALTER TABLE my_database.my_table
   MODIFY SETTING parts_to_throw_insert = 1000,
                 parts_to_delay_insert = 500;
   ```

3. **Check merge progress**
   ```sql
   SELECT * FROM system.merges;
   ```

4. **Increase background merge threads**
   ```xml
   <!-- In config.xml -->
   <background_pool_size>32</background_pool_size>
   <background_merges_mutations_concurrency_ratio>2</background_merges_mutations_concurrency_ratio>
   ```

</Steps>

### Replication Lag

**Check replication status:**
```sql
SELECT 
    database,
    table,
    is_leader,
    absolute_delay,
    queue_size,
    inserts_in_queue,
    merges_in_queue,
    log_max_index - log_pointer AS log_lag
FROM system.replicas
ORDER BY absolute_delay DESC;

-- Check replication queue
SELECT 
    database,
    table,
    type,
    create_time,
    num_tries,
    last_exception
FROM system.replication_queue
WHERE last_exception != ''
ORDER BY create_time;
```

**Solutions:**

1. **Restart replica**
   ```sql
   SYSTEM RESTART REPLICA database.table;
   ```

2. **Sync replica**
   ```sql
   SYSTEM SYNC REPLICA database.table;
   ```

3. **Drop and recreate replica** (last resort)
   ```sql
   -- On lagging replica
   SYSTEM DROP REPLICA 'replica_name' FROM TABLE database.table;
   
   -- Recreate table from another replica
   CREATE TABLE database.table AS database_from_another_replica.table;
   ```

### Mutations Stuck

**Check mutations:**
```sql
SELECT 
    database,
    table,
    mutation_id,
    command,
    create_time,
    parts_to_do_names,
    parts_to_do,
    is_done,
    latest_failed_part,
    latest_fail_reason
FROM system.mutations
WHERE NOT is_done
ORDER BY create_time;
```

**Solutions:**

1. **Kill mutation**
   ```sql
   KILL MUTATION WHERE database = 'my_database' AND table = 'my_table' AND mutation_id = 'mutation_123';
   ```

2. **Check for blocking parts**
   ```sql
   SELECT 
       database,
       table,
       partition,
       name,
       rows
   FROM system.parts
   WHERE (database, table) IN (
       SELECT database, table 
       FROM system.mutations 
       WHERE NOT is_done
   )
   AND active;
   ```

## Connection Issues

### Cannot Connect to Server

**Check server status:**
```bash
# Check if server is running
sudo systemctl status clickhouse-server

# Check listening ports
sudo netstat -tlnp | grep clickhouse

# Test connection
curl http://localhost:8123/
clickhouse-client --query "SELECT 1"
```

**Check firewall:**
```bash
# Ubuntu/Debian
sudo ufw status
sudo ufw allow 8123/tcp
sudo ufw allow 9000/tcp

# CentOS/RHEL
sudo firewall-cmd --list-all
sudo firewall-cmd --add-port=8123/tcp --permanent
sudo firewall-cmd --add-port=9000/tcp --permanent
sudo firewall-cmd --reload
```

### Authentication Failures

**Check user configuration:**
```sql
-- List users
SELECT name, auth_type, host_ip, host_names FROM system.users;

-- Check current user
SELECT currentUser();
```

**Reset password:**
```sql
ALTER USER username IDENTIFIED WITH sha256_password BY 'new_password';
```

## Disk Issues

### Disk Full

**Check disk usage:**
```bash
# Overall disk usage
df -h /var/lib/clickhouse/

# Largest directories
du -sh /var/lib/clickhouse/*/ | sort -rh | head -10
```

**Free up space:**

<Steps>

1. **Drop old partitions**
   ```sql
   -- List partitions
   SELECT DISTINCT partition FROM system.parts WHERE table = 'my_table';
   
   -- Drop old partitions
   ALTER TABLE my_database.my_table DROP PARTITION '202401';
   ```

2. **Clean up temporary files**
   ```bash
   sudo rm -rf /var/lib/clickhouse/tmp/*
   ```

3. **Truncate or drop unused tables**
   ```sql
   TRUNCATE TABLE unused_table;
   DROP TABLE unused_table;
   ```

4. **Clean old logs**
   ```sql
   -- Set retention for query_log
   ALTER TABLE system.query_log MODIFY TTL event_date + INTERVAL 30 DAY;
   OPTIMIZE TABLE system.query_log FINAL;
   ```

</Steps>

### Slow Disk I/O

**Monitor I/O:**
```bash
# Check I/O wait
iostat -x 1 10

# Monitor disk activity
sudo iotop -o

# Check for slow merges
clickhouse-client --query "SELECT * FROM system.merges"
```

**Solutions:**
- Move to faster storage (SSD/NVMe)
- Reduce merge activity
- Add more disks for parallelism
- Optimize table structure

## Diagnostic Queries

### Health Check

```sql
-- Server uptime
SELECT uptime() AS server_uptime_seconds;

-- Version
SELECT version();

-- Current queries
SELECT count() FROM system.processes;

-- Recent errors
SELECT 
    event_time,
    user,
    exception,
    query
FROM system.query_log
WHERE type = 'ExceptionWhileProcessing'
  AND event_date = today()
ORDER BY event_time DESC
LIMIT 10;

-- Replication health
SELECT 
    database,
    table,
    is_readonly,
    absolute_delay,
    queue_size
FROM system.replicas
WHERE absolute_delay > 60 OR is_readonly;
```

### Performance Diagnostics

```sql
-- Top tables by size
SELECT 
    database,
    table,
    formatReadableSize(sum(bytes_on_disk)) AS size,
    sum(rows) AS rows,
    count() AS parts
FROM system.parts
WHERE active
GROUP BY database, table
ORDER BY sum(bytes_on_disk) DESC
LIMIT 20;

-- Current resource usage
SELECT 
    metric,
    value
FROM system.metrics
WHERE metric IN (
    'Query',
    'Merge', 
    'MemoryTracking',
    'BackgroundPoolTask'
);
```

   - Configuration: `/etc/clickhouse-server/`

   - System table snapshots (processes, merges, etc.)

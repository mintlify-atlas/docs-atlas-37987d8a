# Quickstart Guide

Get ClickHouse running and execute your first query in under 5 minutes.

## Installation

The fastest way to start is with the one-line install script:

<CodeGroup>
```bash Linux/macOS/FreeBSD
curl https://clickhouse.com/ | sh
```

```bash Docker
docker run -d --name clickhouse-server \
  --ulimit nofile=262144:262144 \
  -p 8123:8123 -p 9000:9000 \
  clickhouse/clickhouse-server
```
</CodeGroup>

<Note>
The Docker command exposes:
- Port `8123` for HTTP interface
- Port `9000` for native client protocol
</Note>

## Connect to ClickHouse

Once installed, connect using the ClickHouse client:

<CodeGroup>
```bash Native Install
clickhouse-client
```

```bash Docker
docker exec -it clickhouse-server clickhouse-client
```
</CodeGroup>

You should see the ClickHouse prompt:

```
ClickHouse client version 26.2.1.1.
Connecting to localhost:9000 as user default.
Connected to ClickHouse server version 26.2.1.

clickhouse-server :)
```

## Create Your First Table

Create a table using the `MergeTree` engine, ClickHouse's most powerful table engine:

```sql
CREATE TABLE events (
    event_time DateTime,
    event_date Date,
    user_id UInt64,
    event_type String,
    country String,
    revenue Float32
) ENGINE = MergeTree
ORDER BY (event_date, user_id);
```

<Note>
The `MergeTree` engine is the workhorse of ClickHouse, providing:
- Fast inserts
- Efficient compression
- Primary key indexing via `ORDER BY`
- Background data merging and optimization
</Note>

### Analyze Events by Type

```sql
SELECT 
    event_type,
    COUNT(*) AS total_events,
    COUNT(DISTINCT user_id) AS unique_users
FROM events
GROUP BY event_type
ORDER BY total_events DESC;
```

### Calculate Revenue by Country

```sql
SELECT 
    country,
    COUNT(*) AS events,
    SUM(revenue) AS total_revenue,
    AVG(revenue) AS avg_revenue
FROM events
GROUP BY country
ORDER BY total_revenue DESC;
```

### Time-Based Analysis

```sql
SELECT 
    toStartOfHour(event_time) AS hour,
    COUNT(*) AS events
FROM events
GROUP BY hour
ORDER BY hour;
```

## Query via HTTP Interface

You can also query ClickHouse using HTTP with `curl`:

```bash
echo "SELECT COUNT(*) FROM events" | \
  curl 'http://localhost:8123/' --data-binary @-
```

<Tip>
The HTTP interface is perfect for integrations with applications, BI tools, and monitoring systems.
</Tip>

## Using the `numbers` Function

ClickHouse provides built-in table functions for generating test data. The `numbers` function is incredibly useful:


Generate millions of rows for testing:

```sql
INSERT INTO events 
SELECT 
    now() - INTERVAL number SECOND AS event_time,
    toDate(now() - INTERVAL number SECOND) AS event_date,
    number AS user_id,
    ['page_view', 'click', 'purchase'][number % 3 + 1] AS event_type,
    ['US', 'UK', 'FR', 'DE', 'JP'][number % 5 + 1] AS country,
    if(number % 10 = 0, rand() % 200, 0) AS revenue
FROM numbers(100000)
SETTINGS max_insert_block_size = 1000;
```

<Note>
This generates 100,000 realistic events in seconds. Try querying this larger dataset!
</Note>

Standard aggregate functions perform common statistical and mathematical operations. These functions are implemented in `~/workspace/source/src/AggregateFunctions/`.

## Counting Functions

<Accordion title="count">

**Signature**: `count([expr])`

**Source**: `src/AggregateFunctions/AggregateFunctionCount.cpp:254`

Counts the number of rows or not-NULL values.

**Syntax variants**:
- `count()` or `count(*)` - count all rows
- `count(expr)` - count non-NULL values
- `COUNT(DISTINCT expr)` - count distinct values (uses `uniqExact` by default)

**Return type**: `UInt64`

**Examples**:

```sql
-- Count all rows
SELECT count() FROM users;
в”Ңв”Җcount()в”Җв”җ
в”Ӯ    1523 в”Ӯ
в””в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”ҳ

-- Count non-NULL values
SELECT count(email) FROM users;
в”Ңв”Җcount(email)в”Җв”җ
в”Ӯ         1450 в”Ӯ
в””в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”ҳ

-- Count distinct values
SELECT COUNT(DISTINCT country) FROM users;
в”Ңв”ҖuniqExact(country)в”Җв”җ
в”Ӯ                 42 в”Ӯ
в””в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”ҳ
```

**Performance notes**:
- `count()` on MergeTree tables uses metadata (controlled by `optimize_trivial_count_query`)
- `count(nullable_column)` can be optimized with `optimize_functions_to_subcolumns = 1`
- `COUNT(DISTINCT)` implementation set by `count_distinct_implementation` setting

</Accordion>

## Sum Functions

<Accordion title="sum">

**Signature**: `sum(num)`

**Source**: `src/AggregateFunctions/AggregateFunctionSum.cpp:73`

Calculates the sum of numeric values.

**Arguments**:
- `num` - Column of numeric values: `(U)Int*`, `Float*`, `Decimal*`

**Return type**: Automatically promoted to prevent overflow
- `Int8`/`Int16`/`Int32` вҶ’ `Int64`
- `UInt8`/`UInt16`/`UInt32` вҶ’ `UInt64`
- `Float32` вҶ’ `Float32`
- `Float64` вҶ’ `Float64`
- `Decimal32`/`Decimal64` вҶ’ `Decimal128`
- `Decimal128`/`Decimal256` вҶ’ `Decimal256`

**Example**:

```sql
CREATE TABLE employees (
    id UInt32,
    name String,
    salary UInt32
) ENGINE = Memory;

INSERT INTO employees VALUES
    (87432, 'John Smith', 45680),
    (59018, 'Jane Smith', 72350),
    (20376, 'Ivan Ivanovich', 58900),
    (71245, 'Anastasia Ivanovna', 89210);

SELECT sum(salary) FROM employees;
в”Ңв”Җsum(salary)в”Җв”җ
в”Ӯ      266140 в”Ӯ
в””в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”ҳ
```

</Accordion>

<Accordion title="sumWithOverflow">

**Signature**: `sumWithOverflow(num)`

**Source**: `src/AggregateFunctions/AggregateFunctionSum.cpp:173`

Computes sum using the same data type for result as input. If overflow occurs, wraps around.

**Arguments**:
- `num` - Column of numeric values: `(U)Int*`, `Float*`, `Decimal*`

**Return type**: Same as input type

**Example**:

```sql
CREATE TABLE employees (
    id UInt32,
    name String,
    monthly_salary UInt16
) ENGINE = Memory;

INSERT INTO employees VALUES
    (1, 'John', 20000),
    (2, 'Jane', 18000),
    (3, 'Bob', 12000),
    (4, 'Alice', 10000),
    (5, 'Charlie', 8000);

SELECT
    sum(monthly_salary) AS no_overflow,
    sumWithOverflow(monthly_salary) AS overflow,
    toTypeName(no_overflow),
    toTypeName(overflow)
FROM employees;
в”Ңв”Җno_overflowв”Җв”¬в”Җoverflowв”Җв”¬в”ҖtoTypeName(no_overflow)в”Җв”¬в”ҖtoTypeName(overflow)в”Җв”җ
в”Ӯ       68000 в”Ӯ     2464 в”Ӯ UInt64                  в”Ӯ UInt16               в”Ӯ
в””в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”ҙв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”ҙв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”ҙв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”ҳ
```

</Accordion>

<Accordion title="sumKahan">

**Signature**: `sumKahan(x)`

**Source**: `src/AggregateFunctions/AggregateFunctionSum.cpp:206`

Calculates sum using [Kahan compensated summation algorithm](https://en.wikipedia.org/wiki/Kahan_summation_algorithm) for improved floating-point precision.

**Arguments**:
- `x` - Input value: `Integer`, `Float`, `Decimal`

**Return type**: `Float64` for floating-point inputs, appropriate integer type otherwise

**Performance**: Slower than `sum`, but more accurate for floating-point.

**Example**:

```sql
SELECT sum(0.1), sumKahan(0.1) FROM numbers(10);
в”Ңв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җsum(0.1)в”Җв”¬в”ҖsumKahan(0.1)в”Җв”җ
в”Ӯ 0.9999999999999999 в”Ӯ             1 в”Ӯ
в””в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”ҙв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”ҳ
```

</Accordion>

## Average Functions

<Accordion title="avg">

**Signature**: `avg(x)`

**Source**: `src/AggregateFunctions/AggregateFunctionAvg.cpp:91`

Calculates the arithmetic mean.

**Arguments**:
- `x` - Input values: `(U)Int*`, `Float*`, `Decimal`, `Date`, `DateTime`

**Return type**: 
- `Float64` for numeric types
- Same type for `Date`, `DateTime` (rounded to nearest integer)

**Example**:

```sql
SELECT avg(x) FROM VALUES('x Int8', 0, 1, 2, 3, 4, 5);
в”Ңв”Җavg(x)в”Җв”җ

в””в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”ҳ

CREATE TABLE test (t UInt8) ENGINE = Memory;
SELECT avg(t) FROM test;
в”Ңв”Җavg(t)в”Җв”җ
в”Ӯ    nan в”Ӯ
в””в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”ҳ
```

</Accordion>

<Accordion title="avgWeighted">

**Signature**: `avgWeighted(value, weight)`

**Source**: `src/AggregateFunctions/AggregateFunctionAvgWeighted.cpp`

<Accordion title="any / anyLast">

**Signature**: `any(column)`, `anyLast(column)`

**Source**: `src/AggregateFunctions/AggregateFunctionAny.cpp`

Returns an arbitrary value from the group. `any` returns the first encountered, `anyLast` returns the last.

**Arguments**:
- `column` - Column name

**Return type**: Same as input type

**Note**: The result is non-deterministic in distributed queries.

**Example**:


## Statistical Functions

<Accordion title="varPop / varSamp">

**Signature**: `varPop(x)`, `varSamp(x)`

**Source**: `src/AggregateFunctions/AggregateFunctionStatistics.cpp`

Calculates population variance (`varPop`) or sample variance (`varSamp`).

**Arguments**:
- `x` - Numeric values

**Return type**: `Float64`

**Formula**:
- `varPop` = `ОЈ(x - mean)ВІ / n`
- `varSamp` = `ОЈ(x - mean)ВІ / (n - 1)`

</Accordion>

<Accordion title="stddevPop / stddevSamp">

**Signature**: `stddevPop(x)`, `stddevSamp(x)`

**Source**: `src/AggregateFunctions/AggregateFunctionStatistics.cpp`

Calculates population standard deviation or sample standard deviation.

**Return type**: `Float64`

**Formula**:
- `stddevPop` = `sqrt(varPop(x))`
- `stddevSamp` = `sqrt(varSamp(x))`

</Accordion>

## Distinct Counting Functions

<Accordion title="uniq">

**Signature**: `uniq(x)`, `uniq(x, y, ...)`

**Source**: `src/AggregateFunctions/AggregateFunctionUniq.cpp`

Approximately counts distinct values using HyperLogLog algorithm.

**Arguments**:
- One or more expressions

**Return type**: `UInt64`

**Accuracy**: ~2% error for typical cardinalities

**Example**:

```sql
SELECT uniq(user_id) FROM visits;
SELECT uniq(user_id, session_id) FROM events;
```

</Accordion>

<Accordion title="uniqExact">

**Signature**: `uniqExact(x)`, `uniqExact(x, y, ...)`

**Source**: `src/AggregateFunctions/AggregateFunctionUniq.cpp`

Exactly counts distinct values using a hash table.

**Arguments**:
- One or more expressions

**Return type**: `UInt64`

**Memory**: O(n) where n is the number of distinct values

</Accordion>

<Accordion title="uniqCombined">

**Signature**: `uniqCombined(HLL_precision)(x)`

**Source**: `src/AggregateFunctions/AggregateFunctionUniqCombined.cpp`

Counts distinct values using combination of array, hash table, and HyperLogLog.

**Arguments**:
- `HLL_precision` - Optional HyperLogLog precision (9-18, default 17)
- `x` - Expression

**Return type**: `UInt64`

**Performance**: Better than `uniq` for many use cases

</Accordion>

Integration engines allow ClickHouse to query external data sources and integrate with message queues. These engines enable real-time data ingestion, federated queries, and ETL pipelines.

## MySQL

Query MySQL databases directly from ClickHouse (`src/Storages/StorageMySQL.h:24-26`).

### Syntax

```sql
CREATE TABLE mysql_table (
    id UInt64,
    name String,
    created DateTime
) ENGINE = MySQL(
    'host:port',
    'database',
    'table',
    'user',
    'password'
);
```

- **ssl_ca**: SSL certificate authority (optional)

### Use Cases

- Migrate data from MySQL to ClickHouse
- Join ClickHouse data with MySQL tables
- Real-time analytics on MySQL data
- Hybrid OLTP/OLAP queries

### Example

```sql
CREATE TABLE users_mysql (
    user_id UInt64,
    username String,
    email String,
    created_at DateTime
) ENGINE = MySQL(
    'mysql-server:3306',
    'production',
    'users',
    'clickhouse_user',
    'secure_password'
);

-- Query MySQL data from ClickHouse
SELECT 
    toYYYYMM(created_at) AS month,
    count() AS new_users
FROM users_mysql
WHERE created_at >= today() - INTERVAL 30 DAY
GROUP BY month;

-- Join with ClickHouse table
SELECT 
    u.username,
    count(e.event_id) AS event_count
FROM users_mysql u
JOIN clickhouse_events e ON u.user_id = e.user_id
GROUP BY u.username
ORDER BY event_count DESC
LIMIT 10;
```

```sql
    customer_id UInt64,
```

```sql

    '_id'
```

### Use Cases

- Analytics on MongoDB data
- Data migration from MongoDB
- Real-time reporting
- Cross-database joins

### Example

```sql
CREATE TABLE events_mongo (
    _id String,
    user_id UInt64,
    event_type String,
    timestamp DateTime,
    metadata String
) ENGINE = MongoDB(
    'mongodb://mongo-server:27017/analytics',
    'events',
    '_id'
);

-- Query MongoDB from ClickHouse
SELECT 
    event_type,
    count() AS event_count,
    uniq(user_id) AS unique_users
FROM events_mongo
WHERE timestamp >= today() - INTERVAL 7 DAY
GROUP BY event_type
ORDER BY event_count DESC;
```

### Example: Real-time Analytics

```sql
-- 1. Create Kafka engine table
CREATE TABLE events_kafka (
    event_time DateTime,
    event_type String,
    user_id UInt64,
    properties String
) ENGINE = Kafka()
SETTINGS 
    kafka_broker_list = 'kafka1:9092,kafka2:9092',
    kafka_topic_list = 'events',
    kafka_group_name = 'clickhouse_events',
    kafka_format = 'JSONEachRow',
    kafka_num_consumers = 4,
    kafka_thread_per_consumer = 1;

-- 2. Create target MergeTree table
CREATE TABLE events (
    event_time DateTime,
    event_type String,
    user_id UInt64,
    properties String
) ENGINE = MergeTree()
PARTITION BY toYYYYMMDD(event_time)
ORDER BY (event_time, user_id);

-- 3. Create materialized view for ingestion
CREATE MATERIALIZED VIEW events_consumer TO events AS
SELECT 
    event_time,
    event_type,
    user_id,
    properties
FROM events_kafka;

-- Data flows: Kafka -> events_kafka -> events_consumer -> events
```
